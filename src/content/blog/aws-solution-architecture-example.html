Overview

Establishing a solid foundation on AWS is critical for long-term scalability and security. A Multi-Account Landing Zone provides the governance and isolation needed for enterprise workloads.

Architect's Note

While AWS Control Tower automates much of this, understanding the underlying IAM Policy (SCP) structure and Transit Gateway flow is essential for troubleshooting complex networking scenarios in highly regulated environments.

Architecture Diagram

Control Tower vs. Manual Setup

Feature AWS Control Tower Manual (CloudFormation/Terraform)
Setup Speed Fast (Click-to-deploy) Slow (Weeks of development)
Maintenance Automated updates via AWS In-house lifecycle management
Customization Medium (Customizations for Control Tower) Infinite (Total control)
Governance Built-in SCPs and Guardrails Manually authored policies

Core Service Control Policy (SCP)

A typical high-security SCP to prevent users from disabling CloudTrail or deleting S3 logs:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyDisablingCloudTrail",
      "Effect": "Deny",
      "Action": [
        "cloudtrail:StopLogging",
        "cloudtrail:UpdateTrail",
        "cloudtrail:DeleteTrail"
      ],
      "Resource": "*"
    }
  ]
}
